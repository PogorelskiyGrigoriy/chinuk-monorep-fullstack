# 🗺️ Карта функциональности интерфейса "Chinook Explorer"

Этот документ описывает структуру таблиц и логику взаимодействия (модальные окна) для каждой страницы приложения согласно ТЗ.

---

## 1. 👥 Страница: Customers (Клиенты)
**Доступ:** `SALE`, `SUPER_USER` (ТЗ 1.1)

### 📊 Основная таблица
| Колонка | Данные (ТЗ 1.1.1.1) | Действие |
| :--- | :--- | :--- |
| **Имя** | `firstName` | — |
| **Фамилия** | `lastName` | — |
| **Город** | `city` | — |
| **Страна** | `country` | — |
| **Email** | `email` | — |
| **Счета** | Кнопка **"Invoices"** | Открывает **Модальное окно №1** |
| **Менеджер** | Кнопка **"Sales Agent"** | Открывает **Модальное окно №2** |

### 🪟 Вызываемые модальные окна

#### **№1: Таблица инвойсов (Invoices)**
* **Логика:** Список всех счетов выбранного клиента (ТЗ 1.1.1.2.1).
* **Состав строк:** Данные об инвойсе (ID, Дата, Сумма).
* **Кнопка в строке:** **"Details"** — открывает **Модальное окно №3** (Список треков).

#### **№2: Карточка менеджера (Sales Agent)**
* **Логика:** Данные сотрудника, закрепленного за клиентом (ТЗ 1.1.1.2.2).
* **Поля (ТЗ 1.1.1.2.2.1-7):** Имя, Фамилия, Дата рождения, Дата найма, Город, Страна, Email.

#### **№3: Состав инвойса (Track List)**
* **Логика:** Перечень треков конкретного счета (ТЗ 1.1.1.2.1.1).
* **Таблица:** Название трека, Жанр, Тип медиа.

## 4. 🛡️ Страница: Audit (Логи Аудита)
**Доступ:** `SUPER_USER` (ТЗ 2.2 / 2.5.4)

### 📊 Таблица событий (Accounting)
| Колонка | Описание |
| :--- | :--- |
| **ID** | UUID события |
| **Время** | Таймстамп (ISO 8601) |
| **Сотрудник** | ID или Email инициатора |
| **Действие** | Тип события (LOGIN, READ, и т.д.) |
| **Ресурс** | К какому эндпоинту был доступ |
| **Метаданные** | Доп. информация (IP/Браузер) |

---

### 💡 Примечания по разработке:
* **Универсальность:** Все таблицы треков (окна №3, №4, №5) используют идентичный формат данных.
* **Навигация:** Модальное окно Sales Agent — статичное. Окно Invoices поддерживает вложенный вызов треков.
